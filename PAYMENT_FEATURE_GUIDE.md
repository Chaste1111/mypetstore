# 支付功能实现说明

## 📋 功能概述

本次更新实现了完整的订单支付功能，包括：
1. ✅ 创建支付页面，支持三种支付方式（云闪付、微信支付、支付宝）
2. ✅ 从订单详情页和订单列表页跳转到支付页面
3. ✅ 支付成功后订单状态从"待支付"变为"配送中"
4. ✅ 提交订单后自动移除购物车中的对应商品

## 🎯 主要变更

### 前端变更

#### 1. 新增文件
- `frontend/src/views/order/Payment.vue` - 支付页面
- `PAYMENT_IMAGES_GUIDE.md` - 支付图片资源说明

#### 2. 修改文件
- `frontend/src/router/index.js` - 添加支付页面路由 `/order/:orderId/payment`
- `frontend/src/views/order/OrderDetail.vue` - 修改支付按钮为跳转到支付页面
- `frontend/src/views/order/OrderList.vue` - 修改支付按钮为跳转到支付页面

#### 3. 支付页面功能
- **三种支付方式选择**：
  - 云闪付/银行卡支付（默认）
  - 微信支付（显示二维码）
  - 支付宝支付（显示二维码）

- **支付流程**：
  1. 用户选择支付方式
  2. 点击"确认支付"
  3. 根据不同方式显示不同界面：
     - 云闪付：弹出提示框模拟跳转
     - 微信/支付宝：显示二维码对话框
  4. 用户确认支付后，订单状态更新为"配送中"
  5. 跳转到订单详情页

### 后端变更

#### 1. 修改文件
- `src/main/java/csu/web/mypetstore/service/impl/OrderServiceImpl.java`
  - 添加CartService依赖注入
  - 在placeOrder方法中添加移除购物车商品的逻辑
  - 在getOrderById方法中加载lineItems

- `src/main/java/csu/web/mypetstore/persistence/OrderDao.java`
  - 添加getLineItemsByOrderId方法

- `src/main/resources/mapper/OrderMapper.xml`
  - 添加getLineItemsByOrderId的SQL查询

#### 2. 已有接口
- `PUT /api/orders/{orderId}/status` - 已支持更新订单状态
  - 支持状态：待支付(P) → 配送中(S)
  - convertToDbStatus方法已正确处理状态转换

## 📊 订单状态流转

```
待支付 (P) 
    ↓ [用户支付]
配送中 (S)
    ↓ [管理员发货/用户确认收货]
已完成 (C)

或者：

待支付 (P)
    ↓ [用户取消]
已取消 (X)
```

## ⚠️ 注意事项

1. **图片资源**：支付功能需要图片资源才能正常显示，请参考`PAYMENT_IMAGES_GUIDE.md`准备图片

2. **事务处理**：
   - 订单提交时，购物车商品移除在同一个事务中
   - 如果移除失败，会记录警告但不影响订单提交

3. **状态管理**：
   - 前端使用中文状态显示（待支付、配送中等）
   - 后端使用单字符状态码存储（P、S、C、X）
   - 转换逻辑在OrderController的convertToDbStatus方法中

4. **安全性**：
   - 订单查询都带有用户名验证，防止越权访问
   - 支付前会检查订单是否已经支付或取消

5. **用户体验**：
   - 云闪付支付有1.5秒延迟模拟跳转
   - 微信/支付宝支付需要用户手动点击"我已支付"
   - 支付成功后自动跳转到订单详情页
